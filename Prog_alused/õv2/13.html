Erind (exception)

Erind on mingi eriolukord, mis programmi töö jooksul ilmneb. Üldjuhul programmi töö seiskub erindi töttu. Selleks, et programmi töö ootamatult ei lõppeks, tuleb erindeid töödelda, ehk programmile öelda, mida erindi puhul ette võtta.

Erindite töötlemine

Erindeid on võimalik "kinni püüda" ja määrata programmi edasine käitumine. Seda nimetatakse erinditega tegelemisesks (exception handling). Tegelemiseks kasutatakse kahte koodiplokki, try ja except.

try osas lisatakse kood, mis võib tekitada viga. Vea tekkimisel käivitatakse tegevus, mis kirjutatakse except ossa. Juhul kui viga ei teki, siis try osas täidetakse kogu kood. Veel võib lisada ka finally ploki, ei ole kohustuslik, see osa käivitub igaljuhul.

try:
# Try to to something, which may cause an error

except <Name of the error class>
# If what you tried caused an error, this code block is executed
# If the error was not thrown, this code block is ignored

finally:
# This code block is executed anyway

Ilma erindeid "kinni püüdmata" peaksime iga võimalike olukorra määratlema tingimuslausetega. Kood oleks pikk ja raskesti loetav. Kasutades erindeid muutub kood paindlikumaks ja paremini loetavaks.

Erindite tõstatamine programmi töö juhtimiseks

Erindite tõstatamine on kasulik siis, kui mingil konkreetsel juhul tuleb kohe programmi töö lõpetada ja näidata kasutajale vea põhjust ja kohta. Tõstatamine käib korraldusega raise millele lisatakse täpsustatud tegevus:

  • SystemExit lõpetab programmi töö
  • Kutsutakse esile mingi vea teade näiteks: ValueError("This is not a word")

user_input = input("Please input a word: ")

# If the input is a number
if word.isdigit(): raise ValueError("This is not a word!") # Throw an error with specified message
# This will never be printed in case of error
print(f"The word is '{user_input}'")

Kasutaja loodud erinditüübid

Erind on objekt, mis kannab endaga kaasas täiendavat infot:

  • klassi nimi - see annab juba infot, mis probleem võis tekkida (nt IndexError)
  • erindi teade (teade, tekkepõhjus, täpsustus vms)
  • erindi tekkimise koht (fail, rida)
  • kogu eelnenud programmi kutsungite ahel (mis meetod kutsus välja meetodi, kus erind tekkis jne) (stack trace)

# Define Python user-defined exceptions
class Error(Exception):
"""Base class for other exceptions"""
pass

class ValueTooSmallError(Error):
"""Raised when the input value is too small"""
pass

class ValueTooLargeError(Error):
"""Raised when the input value is too large"""
pass

# You need to guess this number
number = 10

# User guesses a number until he/she gets it right
while True:
try: i_num = int(input("Enter a number: ")) if i_num < number: raise ValueTooSmallError elif i_num > number:
raise ValueTooLargeError break except ValueTooSmallError: print("This value is too small, try again!") except ValueTooLargeError: print("This value is too large, try again!")
print("Congratulations! You guessed it correctly.")

Võimalikud väljundid:

Enter a number: 12
This value is too large, try again!

Enter a number: 0
This value is too small, try again!

Enter a number: 8
This value is too small, try again!

Enter a number: 10
Congratulations! You guessed it correctly.

Nii luuakse uus alamklass Error(mis laiendab sisseehitatud Exception klassi) ning uut tüüpi erindid ValueTooSmallError ja ValueTooLargeError, mis laiendavad meie loodud Error klassi. Loodud uuele tüübile me mingit sisu lisada ei taha (ta pärib kõik Exception klassi meetodid ja muutujad). Kuna Pythonis me klassi kirjeldust tühjaks ei saa jätta, lisame sinna võtmesõna pass.
