Objektorienteeritud programeerimise (OOP) põhimõtted, klassid ja objektid

OOP on programeerimise stiil, mis kasutab objekte. Python on OOP keel. Kõik asjad Pythonis on objektid.

Objekt

Objekt kirjeldab ära konkreetse loogilse kogumi. See koosneb tavaliselt mitmest väärtusest. Näiteks nagu auto: värv, mark, mudel, pikkus, registrimass jne.

Sõne on objekt. Kui luua uus sõne või muuta sõne väärtus, siis luuakse uus objekt, mille tüüp on str. Sõne "funktsioone" kutsutakse meetoditeks. Klassis kirjeldatud funktsioonid on meetodid.

Järjend on objekt. Muutes järjendi sisu või andes muutujale teise muutuja väärtus, siis jääb objekt samaks.

Klass

Klass kirjeldab struktuuri. See (üldiselt) ei sisalda andmeid. Klass on andmetüüp. Samatüüpi andmed pärinevad ühest klassist. Näiteks maailmas on erinevaid autosi (objekt), kõik kuuluvad ühte klassi "auto": punane auto on auto, roheline auto on auto.

Iga klass on andmetüüp. Näiteks on Pythonis klass str. Iga konkreetne sõne (näiteks "tere") on selle klassi objekt (ehk isend või instants). Ühest klassist saab luua lõpmata palju objekte.

Klassi tegemine:

class Student: pass

Objektide võrdlemine tehakse == võrdlusega. See kontrollib vaikimisi seda, kas nad viitavad samale objektile. Sõne puhul kontrollitakse, kas sisu (sümbolid) on samad.

s1 = Student()
s2 = Student()
s3 = s1

print(s1 == s2)  # False
print(s1 == s3)  # True
print(s2 == s3)  # False

Meetod

Klassis sisalduvaid funktsioone nimetatakse meetoditeks. Kõik objekti meetodid sisaldavad esimest parameetrit self. See viitab isendile.

self ja parameetrid:
class Student: def greet_friend(self, friend_name): print(f"Hello, {friend_name}")
s = Student()
s.greet_friend("Kaia")

Konstruktor

Objekti loomisel pannakse käima eriline meetod ehk konstruktor. Meetod kirjeldatakse: __init__(self). See meetod pannakse käima üks kord objekti loomisel. Konstruktori kirjeldamine ei ole kohustuslik.

Konstruktor peab tagastama None (eraldi return lauset ei kirjutata).

Seda kirjeldataqkse nagu tavaline meetod. Eraldi pole vaja välja kutsuda.

class Student: def __init__(self): print("Initializing student..")
s = Student() # Initializing student..

Student() kutsub Student klassi konstruktori välja.

Objekti muutujad

self viitab loodavale/loodud objektile. Konstruktorisse saab kaasa anda argumente (nagu tavaline funktsioon). Esimene parameeter on alati self. Objekti muutujad on seotud ühe konkreetse objektiga (isendiga). Objekti muutujaid väärtustatakse: self.name = .... Tavaliselt luuakse konstruktoris vajalikud väljad ära. Objekti muutujaid saab teistes objekti meetodites kasutada.

class Student: def __init__(self, name, title): self.name = name
self.title = title
ago = Student("Ago", "Sir")
print(ago.name)

leela = Student("Leela", "Captain")
print(leela.title)

Konstruktori parameeter self viitab loodavale objektile. Parameetrid name ja title salvestatakse objekti muutujateks: self.name ja self.title.

Erilised meetodid

Python võimaldab klassis ära kirjeldada objekti käitumist erinevates olukordades. Näiteks __eq__ meetodit kasutatakse selleks, et võrrelda objekte == võrdlusega. __str__ meetodit kasutatakse, et saada objektist sõne kuju. Lisaks näiteks __add__(self, other) objektide liitmiseks jne. __lt__(self, other) väikem-kui võrdluseks.

Pythonis on eriline meetod __repr__, mille eesmärk on tagastada üheselt mõistetav sõne.

Klassi muutujad

Klassi muutuja kirjeldatakse klassi sees väljaspool meetodeid. Klassi muutujal on üks väärtus läbi terve programmi. Sõltumata sellest, mitu objekti klassist luuakse, klassi muutujal on üks ühine väärtus.

class Doorbell: click_count = 0

def __init__(self): self.click_count = 0
def ring(self): print("Ringing..")
self.click_count += 1
Doorbell.click_count += 1 --------------------------- d1 = Doorbell()
d2 = Doorbell()

for _ in range(10): d1.ring()
for _ in range(4): d2.ring()
print(d1.click_count)  # 10
print(d2.click_count)  # 4
print(Doorbell.click_count)  # 14

Objekti väärtuste pärimine/muutmine

Kapseldamise eesmärk on peita objekti olek maailma eest ning lubada selle muutmine vaid läbi teatud meetodite. Objekti väärtuste muutmiseks kasutatakse tavaliselt getter ja setter meetodeid. See annab võimaluse kontrollida, mida ja kuidas tagastatakse/muudetakse.

class Student: def __init__(self, name): self.name = name
def set_name(self, name): self.name = name
def get_name(self): return self.name

OOP tehnikad

  • Pärimine (inheritance) - alamklass pärib omadused ja meetodid.
  • Kapseldamine (encapsulation) - funktsionaalsus peidetakse.
  • Polümorfism (polymorphism) - alamklass saab meetodeid üle kirjutada.
  • Modulaarsus (modularity) - programm jagatakse iseseisvateks tükkideks.
