Koodistiil

Igas programmeerimiskeeles on pisut erinev koodistiil, seega asudes uut keelt õppima tuleb vaadata üle ka selle koodistiil. Pythoni koodistiili võtmenimeks on "PEP 8" ning docstring'ide koodistiilil "PEP 257".

Mis on koodistiil ja milleks?

  • Kood on korraldused arvutile
  • Arvuti jaoks pole koodistiilil vahet
  • Koodi on vaja hiljem hallata ja täiendada
  • Koodistiil on oluline arendajale

Jälgides koodistiili, muudame oma koodi paremini loetavamaks. See võimaldab hiljem endal või teistel paremini aru saada, mida mingi koodi osa teeb.

Muutujate nimetamine

  • Muutuja nimi väikeste tähtedega
  • Sõnade eraldajana kasutada alakriipsu _
  • Näiteks: best_score, temperature
  • Mitte kasutada lühendeid näide temp, a, b

Funktsioon

  • Nimi väikeste tähtedega
  • Sõnade eraldajana kasutada alakriipsu _
  • Funktsiooni parameetrite arv võiks olla kuni 4
  • Funktsiooni nimi peaks arusaadavalt kirjeldama, mida funktsioon teeb.
  • Pikkus tavaliselt kuni 20 rida

Tühikute kasutamine

  • Operaatori ees ja järel tühik
    • 3 + 4 * 12
    • age = 18 + additional_years
  • Funktsiooni väljakutsel sulu ees tühikut ei ole
    • do_something(12), print(1)

Docstring

Docstring on kommentaar funktsiooni, meetodi, klassi või mooduli (faili) kohta. Kirjeldatakse kolmekordsete jutumärkide """ """. Eesmärk on kirjeldada funktsionaalsust, võib olla ühe- või mitmerealine. Kasutatakse täislauseid, mis lõppevad punktiga.

Docstring faili (mooduli) alguses

Sellega kirjeldatakse ära, mida see fail teeb. Mitmerealise docstringi puhul on esimene rida kokkuvõte, millele järgneb tühi rida.

"""
Collection of helper functions for string operations.

Most of the functions work with only ascii strings.
Check the docstrings for more information.
"""

Üherealine docstring

Võib kasutada, kui kirjeldus on lühike. Lause peab lõppema punktiga.

"""String helper functions."""

Funktsiooni docstring

Kirjutatakse peale funktsiooni definitsiooni:

def absmax(a: int, b: int) -> int: """
Return a max value of two inputs.

Absolute values of the inputs are used when comparing.
Returns original (non-absolute) value.
"""
return max((a, b), key=abs)

Üherealine peab mahtuma ühele reale koos jutumärkidega.

Docstring stiil

Sisu peab olema täislaustena ja esimene sõna käskivas kõneviisis.

  • Ei sobi: Returns sum.
  • Sobib: Return sum.

Üldised stiilinõuded

  • Faili lõpus peab olema tühi rida
  • Funktsiooni ees peab olema 2 tühja rida
  • Mitmerealise docstring'i esimese ja ülejäänud ridade vahel peab olema tühi rida
  • Rea lõpus ei tohi olla üleliigseid tühikuid

Dokumenteerimine: kommentaar

Kommentaarid on koodikirjeldused, mida kompilaator koodilugemisel eirab. Kommentaarid peaksid olema täislaused ning algama suure algustähega. Kommentaare defineeritakse sümboli # abil. Lisades # rea algusesse, eiratakse tervet koodirida. Kui sümbol ei paikne rea alguses, loetakse sellele eelnev kood ikkagi sisse. Kommentaar algab tühikuga ning kui kommentaar rea keskele lisada, peab koodi lõpu ning # sümboli vahele jätma vähemalt 2 tühikut.
