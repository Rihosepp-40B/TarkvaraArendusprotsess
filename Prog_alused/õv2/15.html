Testimine ja silumine

Selleks, et tagada koodi ja süsteemi töökindlus, tuleks seda testida.

Testide kirjutamiseks tuleb eelkõige kindlaks teha, mida peab testitav funktsioon tegema ja mida see peab erinevate argumentide korral tagastama.

Miks testida?

  • Leiame olukorrad, kus kood ei tööta korrektselt
  • Tagab, et ühe vea parandamisel ei teki uut viga mujale
  • Aitab verifitseerida koodi korrektsus/vastavust nõuetele

Mida testida?

  • Tüüpolukorrad (9 / 2 == 4.5)
  • Piirjuhud (9 / 0 == ?)
  • Absurdsed olukorrad (9 / a == ?)
  • Algoritmi efektiivsus
    • Kas on piisavalt kiire
    • Kas kasutatakse õiget andmestruktuuri
  • Erindi- või veatöötlus

Testimisel tehakse funktsioonid, mis kasutab käsklust assert. Selle abil testitakse kas funktsioon annab oodatud tulemuse. Ühe test funktsiooni all võib olla mitu assert käsklust. Antud käsklus vaatab, kas tulemus on True või False. Teise puhul annab veateate "AssertionError".

    Testid:
  1. test mis õnnestub
  2. test komaga lahendused
  3. üks lahend
  4. lahend puudub
  5. nulliga jagamine
  6. Kas lahend on puudu?
