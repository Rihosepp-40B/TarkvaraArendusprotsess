Rekursioon

Rekursioon on olukord, kus defineeritud funktsioon kutsub iseennast välja, et lahendada funktsioonile antud probleemi kergem variant.

Funktsioonis on sabarekursioon siis, kui selle sama funktsiooni väljakutse on viimane tegevus. Iteratiivne meetod tähendab, et kasutatakse tsükleid. Rekursiivset funktsiooni on alati võimalik ümber kirjutada nii, et see oleks iteratiivne.

Rekursioon kirjeldab lasuse: "Rekursioon on rekursioon". Programmeerimises: probleemi lahendamiseks kutsub funktsioon ennast välja.

def empty_vase(flowers_count):​ if flowers_count <= 0:​ print("Done!") # else:​
empty_vase(flowers_count - 1)

def count_elements(list_of_elements):​ if len(list_of_elements) == 0:​ return 0​ return 1 + count_elements(list_of_elements[1:])

def count_a(text):​ if len(text) == 0:​ return 0​ first_char = text[0]​​
rest_of_the_text = text[1:]​​
first_char_result = 0​​
if first_char == 'a':​ first_char_result = 1​ return first_char_result + \ count_a(rest_of_the_text) ​
Sama tulemus:​
​
def count_a(text):​ if len(text) == 0: return 0​​
return (1 if text[0] == 'a' else 0) + \ count_a(text[1:])

Rekursioon:

  • Funktsioon kutsub ennast välja
  • Funktsioonil peab olema lõpetamistingimus
  • Probleem jagatakse väiksemateks (alam)ülesanneteks
  • Kõiki rekursiivseid probleeme saab lahendada iteratiivselt (ja vastupidi)

Faktoriaal

Faktoriaal arvust on kõikide arvude korrutis alates 1-st kuni selle arvuni.

Sabarekursioon

Sabarekursioon (tail recursion) - funktsioon tagastab rekursiivse väljakutse tulemuse (see on viimane samm).

Rekursioon (võtab rohkem aega, et lahendada): def fibo(n): if n < 3: return 1 return fibo(n - 1) + fibo(n - 2) print(fibo(11)) Iteratiivne lahendus:​​
def fibo_iterative(n):​ fib = 1​​​
prev = 1​​​
for _ in range(n - 2):​ fib += prev​​​
prev = fib - prev​ return fib​ ​
print(fibo_iterative(11)) # 89 ​​
​
Sabarekursioon:​​
def fibo_tail(n, fib, prev):​ if n == 1: return prev​​
return fibo_tail(n - 1, fib + prev, fib) ​​
​
print(fibo_tail(11, 1, 1))  # 89 Sabarekursioonil ja rekursioonil asendada 11 arvuga 1000 annab tulemus errori (maximum recursion depth exceeded in comparison). Tavaline rekursioon võtab kauem aega.

Kokkuvõte

  • Rekursiivne funktsioon kutsub ennast välja
  • Rekursiooni sammu jooksul lahendatakse alamülesanne
  • Rekursioonil peab olema lõpetamistingimus
  • Kõiki rekursiivseid programme saab kirjutada ka iteratiivselt (ja vastupidi)
