Funktsioon

Mis on funktsioon?

  • Funktsiooniga saab kirjeldada alamprogrammi või alamülesannet
  • See on koodiosa, mis täidab mingit konkreetset ülesannet või arvutab midagi
  • Funktsioonil on (üldiselt) nimi

Funktsiooni kirjeldamine

Funktsiooni tähistatakse märkega def millele järgneb nimi sulud ja koolon.

def hello(): print("Hello world!")
Funktsiooni nimi on hello. Funktsiooni sisu on "Hello world!" printimine.

Funktsiooni defineerides võib kirjeldada parameetrid, millega saab funktsiooni juhtida.

def hello_name(name): print(f"Hello {name}!")

Parameeter name, mida saab funktsiooni sees kasutada muutujana. Funktsiooni sisu sõltub name väärtusest.

Funktsiooni kasutamine

Funktsiooni saab välja kutsuda (käivitada) nimega. Kui definitsioon eeldab argumente (deklareeritud parameetrid), tuleb need kaasa anda.

hello()
hello_name("Guido")

Parameetrid ja argumendid

  • Parameeter on muutuja funktsiooni kirjelduses
  • Argument on väärtus, mis funktsiooni saadetakse
  • Argumendi väärtus saab funktsioonis parameetri (muutuja) väärtuseks

def hello_name(name): print(f"Hello {name}!")
hello_name("Marja-Liis")

name on parameeter, "Marja-Liis on argument

Tagastamine return

Pythonis funktsioon alati tagastab midagi. Tagastada saab return korraldusega. Kui return korraldust pole, tagastab funktsioon None.

def triangle_area(a, h): return a * h / 2

Korraldus return lõpetab funktsiooni täitmise.

return vs print()

  • return peatab funktsiooni ja väljastab väärtuse, mida saab kasutada teises funktsioonis
  • print() on lihtsalt üks korraldus (funktsioon), mida saab funktsioonis kasutada (ja ka mujal), mis kuvab ekraanile väärtuse

Vaikeväärtused ja nimelised argumendid

  • Funktsiooni kirjelduses võib parameetrile määrata vaikeväärtuse
    • kui funktsiooni väljakutsumisel sinna väärtust ei määrata, kasutatakse vaikeväärtust

def greet_animal(name, type="Human"): print(f"Greetings {name} of type {type}")

greet_animal("Mari")  # Greetings Mari of type Human
greet_animal("Mari", "Squirrel")  # Greetings Mari of type Squirrel

  • Parameetrid, kuhu väärtust ei anta, kasutavad vaikeväärtusi
  • *args - võtab vastu suvalise arvu positsioonilisi argumente järjendina
  • **kwargs - võtab vastu suvalise arvu märksõnaga argumente sõnastikuna

Anonüümne funktsioon lambda

Lühikesed ja/või ühekordseks kasutamiseks mõeldud, nimeta funktsioonid. Mida kirjeldatakse kujul: lambda args: [tagastatav tulemus]

  • lambda funktsioonil return korraldust ei kirjutata
  • argumendid ei ole sulgudes

ta = lambda a, h: a * h / 2
print(ta(10, 10))

See on sama väärne kui:

def sama_asi(a, h): return a * h / 2
print(sama_asi(10, 10))

filter() ja map()

  • filter() on filtreerimifunktsioon järjendile, väljastab tulemusesse elemendid mille puhul sisendfunktsioon tagastab väärtuse True

nums = [1, 3, 14, 27, 15, 100, 151, 9, 2]
num = lambda x: x % 3 == 0
filtered = filter(num, nums)
print(list(filtered))  # [3, 27, 15, 9]

  • map() on rakendamisfunktsioon järjendile, väljastab tulemusse elementide ja sisendfunktsiooni tehte tulemuse

nums = [1, 3, 14, 27, 15, 100, 151, 9, 2]
num = lambda x: x % 6
mapped = map(num, nums)
print(list(mapped))  # [1, 3, 2, 3, 3, 4, 1, 3, 2]

Tüübivihjed

Kuigi Pythonis muutujatel pole andmetüüpe, saame funktsiooni definitsioonis anda vihjeid andmetüüpide kohta. See on toetatud alates Python'i versioon 3.5'st. Tüübivihjet kasutab IDE, et anda märku võimalikest vigadest.

Iga parameetri järgi saame kooloniga määrata tüübi. Tagastatava tulemuse andmetüüp märgitakse -> järele.

def greeting(name: str) -> str: return f'Hello {name}'
Parameeter on sõne, tagastatakse ka sõne.

Miks kasutada funktsioone?

  • DRY - don't repeat yourself: ära korda koodi, kasuta funktsiooni
  • Abstraktsioon: programmi jagamine alamprogrammideks, mida võivad lahendada erinevad inimesed
  • Testimine: Igat alamprogrammi saab eraldi testida ja täpsemalt tuvastada vea asukohta

Soovitusi kasutamisel

  • Funktsioon peaks tegema ühte konkreetset asja
  • Funktsioon peaks olema piisavalt lühike/lihtne, et nimega saab öelda ära, mida see teeb
  • Koodi on lihtsam kirjutada kui lugeda
  • Hea programmeerija mõtleb sellele, et tema kood oleks hiljem ka teiste poolt loetav
  • Funktsiooni pikkus võiks jääda 10-15 rea vahele
  • Enne, kui hakkad päriselt ülesannet lahendama (detailselt), proovi paika panna struktuur
  • Kui struktuur paigas, hakka alamülesandeid (funktsioone) lahendama
